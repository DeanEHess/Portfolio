# Portfolio

개인용 자산/포트폴리오 트래커입니다. 빌드나 서버 없이 `index.html` 하나로 동작하는 웹앱입니다.

## 왜 만들었나

본인 계좌뿐 아니라 부모님 계좌, IRP·연금저축펀드 등 운용 중인 계좌 종류가 많다 보니 전체 현황을 한눈에 파악하기 어려웠습니다. 계좌 하나 확인하려고 매번 증권사 앱을 켜고 비밀번호를 입력하는 게 너무 번거로워서, 모든 계좌 현황을 한 화면에서 볼 수 있는 도구가 필요해 직접 만들게 됐습니다.

만들고 나서 얻은 부수 효과도 있습니다. 매수/매도할 때마다 '투자 근거'를 남기도록 설계해두니, 잦은 뇌동매매로 인한 후회가 줄었습니다. 전설적인 투자자 필립 피셔는 이렇게 말했습니다 — "내가 주식을 매수할 때 가졌던 당초의 시각을 바꿀 만한 일이 벌어지지 않았다면, 나는 적어도 3년은 계속 보유할 것이다." 근거를 기록해두면 나중에 그 근거가 정말 바뀌었는지 스스로 되짚어볼 수 있어서, 이 원칙을 지키기가 훨씬 쉬워집니다.

그래서 이 앱은 신규 투자자보다는 **이미 여러 계좌에 투자 중인 사람**에 포커싱되어 있습니다. 계좌를 등록할 때 지금까지의 예수금·보유 종목을 처음부터 다시 입력하지 않고 '기초 예수금'과 '이월 잔고'로 현재 상태 그대로 시작할 수 있습니다.

## 어떻게 작동하는가

- **순수 정적 웹앱**: 빌드/서버 없이 `index.html` 하나로 동작하고, GitHub Pages로 배포됩니다.
- **데이터 저장**: 브라우저의 `localStorage`에 저장됩니다(서버 DB 없음). 여러 프로필을 만들어 구분할 수 있습니다.
- **시세/환율 조회**: Cloudflare Worker(`worker.js`, 별도 관리)가 시세 API를 프록시합니다. 가격을 서버에 캐시하지 않고 매번 실시간으로 조회합니다. 이 Worker는 허용된 사이트 주소에서만 호출할 수 있도록 CORS가 제한되어 있고, 클라우드 백업 저장(`kv-save`)에는 도배성 덮어쓰기를 막기 위한 시간당 쓰기 횟수 제한이 걸려 있습니다.
- **선택적 클라우드 백업**: uid + 암호화 키 조합으로 클라이언트 측에서 AES-GCM 종단간 암호화를 한 뒤 Cloudflare KV에 저장합니다. 이 기능을 켜지 않으면 어떤 데이터도 서버로 전송되지 않습니다.
- **계좌 종류**: 증권계좌(혼합, 원화·달러 동시 지원) / 국내전용(ISA·IRP·연금) / 코인 / 현금, 4가지를 지원합니다. 더 필요한 기능 있으시면 얼마든지 건의 부탁드립니다.
- **핵심 기능**:
  - 매수·매도 시 투자 근거 필수 입력
  - 기초 예수금 설정 (이미 투자 중인 계좌를 현재 상태 그대로 등록)
  - 이월 잔고 (과거 매매기록 없이 현재 보유 수량·평단만 등록)
  - 리밸런싱 목표 비중에 따른 매수 제안
  - 처음 접속 시 안내하는 인터랙티브 튜토리얼
  - 종목별·섹터별 비중 도넛 차트

## 신규 사용자가 미리 알아두면 좋은 점

- **데이터는 이 브라우저·기기에만 저장됩니다.** 브라우저 데이터/캐시를 지우거나 다른 기기·브라우저로 접속하면 데이터가 보이지 않습니다. 클라우드 백업을 켜지 않았다면 '백업' 버튼으로 주기적으로 파일을 받아두는 걸 권장합니다.
- **클라우드 백업 키를 잃어버리면 아무도(개발자 포함) 복구할 수 없습니다.** 종단간 암호화 방식이라 서버는 암호화된 내용만 가지고 있습니다. 발급된 키는 반드시 메모장 등 안전한 곳에 따로 저장해두세요.
- **개인 프로젝트이며 소수 인원이 사용합니다.** 상용 서비스 수준의 안정성이나 지원을 보장하지 않으며, 아직 개선이 진행 중인 도구입니다.
- **투자자문이 아닙니다.** 리밸런싱 제안이나 수익률 계산은 참고용 기록 도구일 뿐이며, 실제 투자 판단과 그 책임은 본인에게 있습니다.
- **시세는 무료 소스를 기반으로 하여 약간의 지연이나 오차가 있을 수 있습니다.** 실시간 체결가와 정확히 일치하지 않을 수 있습니다.
- **모바일/PC 브라우저 모두에서 쓸 수 있도록 반응형으로 만들었지만**, 최신 브라우저 기준으로 테스트되었습니다.
- **클라우드 백업을 켜지 않는 한 어떤 데이터도 외부로 전송되지 않습니다.** 완전히 로컬에서만 동작하는 걸 기본으로 하고, 클라우드 백업을 켜더라도 암호화된 내용만 전송됩니다.
